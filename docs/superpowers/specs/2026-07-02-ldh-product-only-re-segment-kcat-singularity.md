# LDH §5b — Product-only rapid-equilibrium segment: `rate_equation` singularity at zero product

**Date:** 2026-07-02
**Status:** Analysis + ODE-validated diagnosis. **No fix implemented** — the fix
direction is a design decision (see Recommendation). **Re-confirmed live 2026-07-09**
(post-PR #62): the pivot/combined-solve work is unrelated to the division-free segment
reduction, and `rate_equation` still returns `NaN` at products = 0 for mechanism 1 (both
the catalytic core and the allosteric form; finite elsewhere).
**Scope:** 28 of 100,787 fits in the 2026-06-27 LDH `identify_rate_equation` run
(`docs/ldh_hpc_results/2026_06_27_results`). Rare, and reached only on the allosteric path.

## Summary

The 28 mechanisms are valid **mixed steady-state / rapid-equilibrium random bi-bi LDH
models** (including the two real LDH abortive complexes `E·NAD·Pyruvate` and
`E·NADH·Lactate`). They all share one structural feature: the catalytic product form
`E·Lactate·NAD` carries **both** products and lands in a **rapid-equilibrium segment whose
forms are all product-bound** (no product-free reference form). For such a segment the
division-free derivation's per-segment enzyme reference is itself product-bound, so every
numerator and denominator monomial picks up a product factor and the derived `rate_equation`
evaluates to `0/0 = NaN` at products = 0.

**ODE ground truth (validated):** the true forward rate at products = 0 is **finite and
unique** for all 28. `NaN` at products = 0 is therefore a **derivation defect** — the
rapid-equilibrium approximation breaking down for a product-only segment — *not* a
genuinely ill-posed kcat. An earlier analysis reported a "22 path-dependent / 6 unique"
split; that split is a property of the defective *closed form* (an order-of-limits artifact
of the RE approximation), not of the physics. See §"ODE validation".

**Why it matters:** the LDH dataset has **142 of 406 rows at products exactly 0** (forward
initial-velocity measurements). At those rows `rate_equation` is `NaN`, which `loss!`
converts to the sign-mismatch sentinel (+100/point) — flooring these mechanisms' loss at
~35 vs ~0.01 for the good models. Separately, `_kcat_forward` throws outright
("produced no kcat components") and the candidate is **dropped from the search entirely**
(loss/retcode/n_params all `missing`). So the defect silently excludes structurally valid
LDH mechanisms.

## The mechanism (exemplar: mechanism 1 of 28)

Random bi-bi, tetramer (catalytic multiplicity 4), forward `NADH + Pyruvate → Lactate + NAD`.
Kinetic classification (ground truth from the derivation's `is_equilibrium` per kinetic
group): **3 rapid-equilibrium groups, 4 steady-state groups (including catalysis)** — i.e.
6 RE steps, 7 SS steps. It is *not* "all RE except catalysis."

Rapid-equilibrium segments (forms in fast equilibrium):
- **seg 1:** {E, E·Pyruvate}
- **seg 2:** {E·Lactate, E·NAD, E·NAD·Pyruvate, E·Lactate·NAD}  ← **all product-bound (the trigger)**
- **seg 3:** {E·NADH, E·Lactate·NADH, E·NADH·Pyruvate}

Steps (SS = steady-state on/off rates; RE = rapid-equilibrium `Kd`):

| kind | step |
|---|---|
| SS | E → E·Lactate (bind Lactate) |
| SS | E → E·NAD (bind NAD) |
| SS | E·Pyruvate → E·NAD·Pyruvate (bind NAD) |
| SS | E → E·NADH (bind NADH) |
| SS | E·Lactate → E·Lactate·NADH (bind NADH)   ← abortive `E·NADH·Lactate` |
| SS | E·Pyruvate → E·NADH·Pyruvate (bind NADH) |
| RE | E ⇌ E·Pyruvate (bind Pyruvate) |
| RE | E·NAD ⇌ E·NAD·Pyruvate (bind Pyruvate)   ← abortive `E·NAD·Pyruvate` |
| RE | E·NADH ⇌ E·NADH·Pyruvate (bind Pyruvate) |
| RE | E·Lactate ⇌ E·Lactate·NAD (bind NAD) |
| RE | E·NAD ⇌ E·Lactate·NAD (bind Lactate) |
| RE | E·NADH ⇌ E·Lactate·NADH (bind Lactate) |
| **SS** | **E·NADH·Pyruvate → E·Lactate·NAD (catalysis)** |

The forward catalytic complex `E·NADH·Pyruvate` is reachable from free enzyme through
product-free forms (`E → E·Pyruvate → E·NADH·Pyruvate` and `E → E·NADH → E·NADH·Pyruvate`),
so the mechanism is not dead-forward.

## Root cause

In segment 2 every form carries a product, so the derivation's per-segment reference (a
free-enzyme-like form used by the division-free reduction) is product-bound. Concretely,
`E·NAD ⇌ E·Lactate·NAD` and `E·Lactate ⇌ E·Lactate·NAD` couple the two products, giving the
rapid-equilibrium relation `[E·NAD]/[E·Lactate] = (NAD·K)/(Lactate·K')`. `_compute_alpha`
emits `α[E·NAD] ∝ NAD/Lactate`, a negative-product-exponent (Laurent) term. When
`_reduce_conc_lowest_terms` (`src/sym_poly_for_rate_eq_derivation.jl`) multiplies numerator
and denominator by `Lactate` to clear that Laurent term, the 16 product-free constant
monomials shift into product-bearing ones. The reduced denominator then has **no constant
term**, so at products = 0 both numerator and denominator vanish → `0/0 = NaN`; and
`_kcat_forward`'s substrate-only-monomial matcher (`src/rate_eq_derivation.jl:880-884`)
finds nothing → the "no components" error. The structural trigger is precise:
**a rapid-equilibrium segment containing no product-free enzyme form.**

## ODE validation (ground truth)

Method: the package's own mass-action ODE steady-state validator (`ode_steady_state_flux`,
`RadauIIA9`, `abstol/reltol 1e-13`; RE steps at `kf = 1e6`, `kr = 1e6·Kd`; SS steps at real
`kon/koff`), on the **non-allosteric catalytic core** `catalytic_mechanism(em)` of each of
the 28 (which carries the identical defect — see §"Not an allosteric bug"). For each: 5
random-parameter trials at nonzero products, plus products = 0 and rays approaching it.

Findings:
1. **Derivation correct where defined:** `rate_equation` matches the ODE at nonzero products
   for **28/28** (rtol 1e-3; one 4/5 trial was a near-equilibrium tolerance artifact,
   39/40 on re-run).
2. **Physical rate at products = 0 is finite for 28/28**; `rate_equation` is `NaN` for 28/28.
3. **The physical value at products = 0 is unique** (a linear master equation has a unique
   steady-state nullspace, so the flux is direction-independent).
4. **The analytic path-dependence is an RE-approximation artifact.** Side-by-side ODE vs
   analytic on one mechanism (substrates = 5, products → 0):

   | (Lactate, NAD) | ODE flux (physical) | `rate_equation` |
   |---|---|---|
   | (0, 0) exactly | **0.8301** | **NaN** |
   | (1e-3, 1e-6) Lac≫NAD | 0.9583 | 0.9607 |
   | (1e-5, 1e-8) Lac≫NAD | 0.8762 | 0.9608 |
   | (1e-6, 1e-3) NAD≫Lac | 0.7116 | 0.7092 |
   | (1e-8, 1e-5) NAD≫Lac | 0.7898 | 0.7094 |
   | (1e-8, 1e-8) diagonal | 0.8301 | 0.8403 |

   At moderate products the two agree. As products → 0 the analytic freezes at a
   **path-dependent** limit (0.96 / 0.71 / 0.84 — confirmed exact to 400-bit BigFloat,
   stable from ε=1e-9 to 1e-40), while the physical ODE **diverges from it and converges to
   a single value** (0.83). The reason is order of limits: the derivation sends RE rates → ∞
   *first* (a ratio-dependent 0/0), but physically product-binding rate is `kon·[product]`,
   which → 0 as products → 0 regardless of how fast the step is, so the enzyme lands in one
   definite distribution. The rapid-equilibrium idealization of a product-only segment is
   simply invalid at products = 0 (a segment with no product cannot be at product-binding
   equilibrium).

**Conclusion:** `NaN`@products=0 is a derivation defect for all 28. These are valid,
fittable LDH mechanisms whose derived rate law is singular exactly where 142/406 data points
live.

## Not an allosteric bug (reached only on the allosteric path)

All 28 are `AllostericEnzymeMechanism` (17/28 have genuine `:NonequalAI` differentiation;
11/28 are trivially allosteric). But the defect is **catalytic**: the non-allosteric
catalytic core `catalytic_mechanism(em)` — a legitimate standalone random bi-bi
`EnzymeMechanism`, atom-conserving and connected — reproduces the identical `NaN`@0 and
empty-kcat failure (its non-allosteric `_kcat_forward` even lacks the `isempty` guard and
throws a bare `MethodError: no method matching max()`). The allostery is incidental.

Yet the run contains **zero** non-allosteric examples: it fit ~1,894 non-allosteric
8-parameter mechanisms, and none errored (which they would have, via the `MethodError`).
At the param counts where this topology appears (n≥9) the search was 93–99.9% allosteric,
and this specific product-only-RE-segment structure is apparently only *reached* through the
allosteric-expansion moves in the beam. So: a catalytic derivation defect that, in practice,
this search encounters only on the allosteric path. (Whether the non-allosteric enumerator
can produce it standalone was not exhaustively tested.)

## Recommendation (design decision — not yet made)

These mechanisms are biochemically valid with a well-defined finite forward rate at
products = 0; do **not** reject them on validity grounds. Options, in order of preference:

1. **Make the derived rate law finite at products = 0** by handling the product-only-RE
   segment's boundary (e.g. detect the missing constant term and supply the finite
   physical value / take the RE limit with product-binding removed). This is the correct
   fix but is real division-free-derivation work and must preserve the 0-alloc/sub-100 ns
   `rate_equation` contract and match the ODE at the boundary. It also fixes `_kcat_forward`
   for free.
2. **Make `_kcat_forward` non-fatal** regardless: on failure degrade to skipping the
   `scale_k_to_kcat` rescale (fit with absolute params) rather than dropping the candidate,
   and add the missing `isempty` guard on the non-allosteric path (currently a bare
   `MethodError`). This stops silently discarding valid mechanisms even before (1) lands.
3. A reject-topology validator keyed on "rapid-equilibrium segment with no product-free
   reference form" is a **fallback only** — it would exclude valid biology and must be
   proven to reject none of the ~21,700 passing mechanisms.

Given the rarity (28/100,787) and that these are high-parameter candidates the parsimony
beam would likely discard anyway, the urgency is correctness/robustness of the derivation
and `_kcat_forward`, not model quality for this particular dataset.

validate one:

```julia
em = Core.eval(EnzymeRates, Meta.parse(sig_string))()     # the AllostericEnzymeMechanism
cm = EnzymeRates.catalytic_mechanism(em)                  # non-allosteric catalytic core
# rate_equation is NaN at products=0, finite elsewhere:
# ER.rate_equation(cm, concs_with_Lactate_NAD_0, params) -> NaN
# ODE ground truth (test/test_rate_eq_derivation.jl helpers) -> finite, unique
```

Scripts used: session scratchpad `show_ssre.jl` (SS/RE classification), `verify_5b_fit.jl`
(data products=0 count + `loss!`), `ode_arbitrate.jl` (ODE-vs-analytic table above),
`bigfloat_check.jl` (400-bit path-dependence confirmation); the full ODE report is
`5b_ode_report.md`.
